# Asset Naming Spec

## 1. Core Rule

episode id is the only permanent identifier.

title anchors may change. episode ids do not.

## 2. Naming Pattern

story:
- `E001_manuscript.md`

song:
- `E001_song_brief.md`
- `E001_theme_song.md`

visual:
- `E001_visual_cut_list.md`
- `E001_mv_prompt_sheet.md`

support:
- `E001_episode_harness.md`

## 3. Cross-Media Rule

story, song, and mv for the same episode must always share the same episode id.

## 4. Public Title Rule

- internal work can use title anchor
- public release can use localized or revised episode titles
- the asset system should still point back to episode id
