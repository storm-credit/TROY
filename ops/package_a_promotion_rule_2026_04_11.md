# package A promotion rule 2026-04-11

## 1. purpose

- 이 문서는 `package A first wave` 반응 이후 언제 `package B` 또는 `package C`로 올릴지 잠근다
- 목표:
  - 한 명의 인상적인 반응에 과속하지 않는다
  - 서로 다른 reader type의 신호를 비교한다
  - promotion과 hold 기준을 명확히 한다

## 3. promote to package B when

- 서로 다른 reader type 2명 이상이 공통으로
  - 계속 읽고 싶다고 말했거나
  - full manuscript를 보고 싶다는 신호를 줬을 때
- 또는
  - 한 명이 강하게 full manuscript를 요청했고
  - 다른 한 명 이상이 최소한 tone / scene / continue intent를 긍정적으로 확인했을 때
- 승급 후 기록:
  - `export/submission/24_PACKAGE_B_FULL_READ_LOG.md`

## 4. promote to package C when

- 신뢰도 높은 대상이
  - 구체적인 수정 코멘트를 줄 의사가 분명하고
  - `where / why` 기준으로 피드백을 줄 수 있을 때
- 단:
  - first wave 전체가 아직 충분히 돌지 않았는데 무조건 `package C`부터 넓히지는 않는다
- 승급 후 기록:
  - `export/submission/25_PACKAGE_C_FEEDBACK_INTAKE_LOG.md`

## 5. hold when

- 반응이 아직 1명 이하일 때
- 긍정/부정 신호가 reader type마다 너무 갈릴 때
- `좋았다`는 말은 있지만 continue intent나 concrete signal이 약할 때

## 6. close when

- 답이 없고 follow-up 이후에도 회신이 없을 때
- reader type 적합도가 낮아 비교 가능한 신호를 주기 어려울 때

## 7. must-avoid

- 한 명의 강한 호평만으로 `package B` 대량 확장
- 수정 욕구가 커서 첫 반응 수집 전에 `package C`부터 여는 일
- reader type 구분 없이 반응을 섞어 읽는 일

## 8. orchestra note

- package A의 일은 심사 통과가 아니라 `어떤 문이 먼저 열리는지`를 확인하는 것이다
- promotion은 enthusiasm보다 비교 가능한 signal이 모였을 때만 한다
- `type C`는 기본 first-wave lane이 아니라 `package C` 상세 피드백 lane에서 더 유효하다
