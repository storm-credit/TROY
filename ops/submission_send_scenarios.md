# submission send scenarios

## 1. purpose

- 이 문서는 실제 외부 전달 상황을 세 가지로 나눠서 무엇을 보내야 하는지 잠근다
- 목표:
  - showcase only
  - full manuscript review
  - alpha feedback request
  를 섞지 않고 운영한다

## 2. truth source

- `ops/submission_delivery_checklist.md`
- `ops/submission_release_readiness_audit.md`
- `ops/alpha_reader_feedback_intake_opening.md`
- `ops/feedback_triage_protocol.md`
- `export/submission/README.md`

## 3. scenario A - showcase only

- use when:
  - 작품의 대표 샘플만 먼저 보여 주고 싶을 때
- send:
  - `00_START_HERE.md`
  - `01_EVALUATOR_NOTE.md`
  - `showcase_index.md`
  - relevant showcase folders
- avoid:
  - full manuscript 동봉

## 4. scenario B - full manuscript review

- use when:
  - 전체 원고 통독 판단이 필요할 때
- send:
  - `00_START_HERE.md`
  - `01_EVALUATOR_NOTE.md`
  - `showcase_index.md`
  - `너라는운율_submission_draft_v1.md`
- optional:
  - strongest showcase bundle 1개

## 5. scenario C - alpha feedback request

- use when:
  - 실제 읽기 반응과 수정 포인트를 받고 싶을 때
- send:
  - `00_START_HERE.md`
  - `01_EVALUATOR_NOTE.md`
  - `02_ALPHA_READER_FEEDBACK_GUIDE.md`
  - `03_FEEDBACK_TEMPLATE.md`
  - `04_FEEDBACK_TRIAGE_NOTE.md`
  - `05_DELIVERY_CHECKLIST.md`
  - `너라는운율_submission_draft_v1.md`
- optional:
  - `showcase_index.md`
  - selected showcase folders

## 6. director note

- 오케스트라는 목적이 섞이면 전달 품질이 떨어진다고 본다
- 먼저 `무엇을 받고 싶은지`를 정하고 그에 맞는 묶음만 보낸다
