# alpha reader feedback intake opening

## 1. purpose

- 이 문서는 `stable state` 이후 실제 외부 검토 피드백을 받기 위한 intake 구조를 연다
- 목표:
  - 외부 검토자가 무엇을 보면 되는지, 무엇을 기준으로 피드백을 주면 되는지 미리 정렬한다
  - 이후 피드백이 들어왔을 때 원고를 무작정 흔들지 않고 `선택적 수정`으로 연결되게 만든다

## 3. current judgment

- current stack:
  - 외부 검토용으로 충분히 닫혀 있음
- next useful move:
  - 새 wave 추가가 아니라 `feedback intake clarity` 확보
- no-go:
  - 피드백 받기 전 선제적 대수술
  - 평가 기준 없이 취향 메모만 모으는 방식

## 4. preferred feedback lanes

- first-read impression:
  - 처음 3화 안에서 어떤 감각과 거리감이 잡히는지
- relationship readability:
  - 서준/아린 관계 변화가 지나치게 빠르거나 과소설명처럼 읽히지 않는지
- motif clarity:
  - `읽기 vs 묻기`, `과속 해석`, `듣지 못함 이후의 사랑`이 잘 잡히는지
- pacing:
  - 느린데 지루한지, 느린데 긴장감이 유지되는지
- ending ethics:
  - 결말이 과열된 재결합이 아니라 낮고 선명한 종지로 읽히는지

## 5. director note

- intake의 목적은 `좋아요/별로예요` 수집이 아니라, 실제 수정 가능한 판단 단위를 받는 것이다
- 오케스트라는 이후 피드백을 `유지 / 조정 / 보류`로 분류해 다룬다
