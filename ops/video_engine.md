# TROY Video Engine

## 1. Definition

`영상 엔진`은 소설과 곡 사이의 감정을 MV 문법으로 번역하는 하위 시퀀스 엔진이다.

상위 원칙:

- MV is emotional condensation, not plot summary
- shot order carries feeling
- leave residue after the cut

## 2. Function

- 회차별 곡을 MV 구조로 시각 번역한다
- 컷 수, 장면 호흡, 기억 장면의 순서를 통제한다
- 이미지 엔진이 만든 stills를 움직임과 리듬으로 엮는다

## 3. Core Rules

### A. Cut Rule

- baseline: 8-12 cuts per episode
- emotional episodes may go 12-16
- transitions should follow residue, not only beat drops

### B. Story Rule

- do not restage the full episode
- choose memory anchors
- choose one visual thesis per MV

### C. Motion Rule

- gentle push, hold, drift, fade preferred
- avoid flashy edit grammar by default
- movement should feel like remembering

### D. Ending Rule

- most episodes should end on residue, not closure
- final frame should extend the song's feeling

## 4. Inputs

- song brief
- image engine outputs
- episode harness
- relationship thermometer
- motif ledger

## 5. Outputs

- MV cut list
- shot order
- cut purpose
- prompt set for still generation
- carry-over continuity note
