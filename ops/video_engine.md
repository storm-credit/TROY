# TROY Video Engine

## 1. Definition

`영상 엔진`은 소설과 곡 사이의 감정을 MV 문법으로 번역하는 하위 시퀀스 엔진이다.

상위 원칙:

- MV is emotional condensation, not plot summary
- shot order carries feeling
- leave residue after the cut

## 2. Function

- 회차별 곡을 MV 구조로 시각 번역한다
- 컷 수, 장면 호흡, 기억 장면의 순서를 통제한다
- 이미지 엔진이 만든 stills를 움직임과 리듬으로 엮는다

## 3. Core Rules

### A. Cut Rule

- baseline: 8-12 cuts per episode
- emotional episodes may go 12-16
- transitions should follow residue, not only beat drops

### B. Story Rule

- do not restage the full episode
- choose memory anchors
- choose one visual thesis per MV

### C. Motion Rule

- gentle push, hold, drift, fade preferred
- avoid flashy edit grammar by default
- movement should feel like remembering

### D. Ending Rule

- most episodes should end on residue, not closure
- final frame should extend the song's feeling

## 4. Composition Methods

### A. 8-12 Cut Emotional Sequence

기본 MV 포맷은 고정된 감정 슬롯을 가진다.

권장 슬롯:

- hook
- setup
- approach
- break
- hold
- tag

운영 필드:

- cut count
- slot map
- shot scale mix
- text overlay limit

### B. Expectation vs Reality Two-Track

이 작품의 영상은 기대와 현실의 이중 문법을 사용할 수 있다.

운영 필드:

- expectation look
- reality look
- switch point
- max switch count

### C. Memory: Seed -> Distort -> Correct

기억 장면은 예쁘게 재현하는 것이 아니라, 왜곡과 수정까지 포함해야 한다.

운영 필드:

- memory seed cut
- distortion method
- correction method
- reuse limit

### D. Scene Selection Rule

한 컷은 최소한 아래 세 가지 중 둘 이상을 가져야 한다.

- one prop
- one distance cue
- one air cue

운영 필드:

- prop
- distance
- air
- pass/fail

### E. Ending Frame Triad

엔딩 프레임은 여운을 남기는 세 가지 형식으로 관리한다.

예:

- empty seat
- unworn earphones
- river wind

운영 필드:

- ending frame type
- last two seconds hold rule
- overlay line if any
- residue type

## 5. Inputs

- song brief
- image engine outputs
- episode harness
- relationship thermometer
- motif ledger

## 6. Outputs

- MV cut list
- shot order
- cut purpose
- prompt set for still generation
- carry-over continuity note
