# TROY Package A Human Input Checklist

## 1. purpose

이 문서는 `package A first wave`에서 이제 사람이 직접 넣어야 하는 마지막 입력만 따로 모아 보여 준다.

핵심:

- 지금 남은 일은 새 문서를 만드는 것이 아니다
- 실제 사람, 채널, 날짜를 넣는 것이다

## 2. required human inputs

### input A

- `19_PACKAGE_A_ALIAS_MAPPING_SHEET.md`
- 넣어야 하는 것:
  - `Proposed Alias`
  - `Qualification Score`
  - `Preferred Channel`
  - `Approved`

### input B

- `13_PACKAGE_A_FIRST_WAVE_EXECUTION_BOARD.md`
- 넣어야 하는 것:
  - `Contact Alias`
  - `Channel`
  - `Planned Send Window`
  - `Follow-Up Window`

### input C

- `12_PACKAGE_A_FIRST_WAVE_CONTACT_TRACKER.md`
- 발송 직후 넣어야 하는 것:
  - `Send Date`
  - `Status = sent`
  - `Notes`

### input D

- `18_PACKAGE_A_SEND_SESSION_LOG.md`
- 발송 배치 직후 넣어야 하는 것:
  - `Send Date`
  - `Slots Sent`
  - `Channels Used`
  - `Total Count`
  - `Follow-Up Due Window`

## 3. minimal go-live path

1. `19_PACKAGE_A_ALIAS_MAPPING_SHEET.md` 작성
2. `13_PACKAGE_A_FIRST_WAVE_EXECUTION_BOARD.md` 작성
3. `17_PACKAGE_A_SLOT_SEND_CHECKLIST.md` 최종 확인
4. 실제 발송
5. `12_PACKAGE_A_FIRST_WAVE_CONTACT_TRACKER.md` 업데이트
6. `18_PACKAGE_A_SEND_SESSION_LOG.md` 업데이트

## 4. do-not-overthink

- primary 3명부터 시작한다
- secondary slot은 지금 비워 둬도 된다
- perfect candidate를 기다리느라 발송을 미루지 않는다
- 첫 반응은 자료 검증이지 최종 심판이 아니다

## 5. orchestra note

- 현재 시스템은 `go-live ready`
- 남은 것은 운영자의 실제 입력뿐이다
