# TROY Package C Alpha Feedback Handoff

## 1. purpose

이 문서는 `package C`로 넘어가 실제 수정 의견 회수를 요청할 때 무엇을 보내고 무엇을 요청할지 정리한 handoff 문서다.

## 2. use when

- 상대가 상세 피드백 의사가 있다
- `where / why` 기준의 코멘트를 줄 수 있다
- 단순 감상보다 수정 후보 수집이 목적이다

## 3. send bundle

- `00_START_HERE.md`
- `01_EVALUATOR_NOTE.md`
- `02_ALPHA_READER_FEEDBACK_GUIDE.md`
- `03_FEEDBACK_TEMPLATE.md`
- `04_FEEDBACK_TRIAGE_NOTE.md`
- `05_DELIVERY_CHECKLIST.md`
- `너라는운율_submission_draft_v1.md`

## 4. what to ask

- 초반 1-3화 진입감
- 중반부 반복/늘어짐 구간
- 결말 온도 반응
- 어디서 왜 그렇게 읽혔는지

## 5. message stance

- 상세 코멘트는 요청하되 강압적으로 느껴지지 않게 한다
- 길이보다 위치와 이유가 중요하다고 안내한다
- 모든 의견을 반영하는 것이 아니라는 점을 운영 문서로 흡수한다

## 6. must-avoid

- 첫 접촉부터 package C 사용
- 너무 많은 질문으로 피로를 주는 일
- 작품 방향을 무시한 자극성 요구를 무비판적으로 수용하는 일

## 7. next step

- 회신이 오면:
  - `03_FEEDBACK_TEMPLATE.md` 또는 별도 intake log로 수집
- triage는:
  - `04_FEEDBACK_TRIAGE_NOTE.md`
  - `ops/feedback_triage_protocol.md`
  기준으로 처리한다
