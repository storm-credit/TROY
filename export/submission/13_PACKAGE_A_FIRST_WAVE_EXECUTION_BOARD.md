# TROY Package A First-Wave Execution Board

## 1. how to use

이 문서는 `package A first wave`를 실제 발송 직전 상태로 정리하는 실행 보드다.

사용 순서:

1. `Contact Alias`에 실제 사람을 식별 가능한 내부 별칭으로 적는다
2. `Channel`에 보낼 채널을 적는다
3. `Planned Send Window`를 정한다
4. 발송 후 `Tracker Slot`과 함께 `12_PACKAGE_A_FIRST_WAVE_CONTACT_TRACKER.md`를 갱신한다

원칙:

- 첫 파동은 primary 3명부터 시작한다
- secondary 3명은 즉시 발송하지 않고 대기 상태로 둔다
- 실명 대신 내부 별칭만 기록한다

## 2. recommended cadence

- day 0:
  - `A-primary-01`, `B-primary-01`, `D-primary-01` 발송
- day 3-5:
  - 열람/답장 여부 확인
- day 7:
  - 무응답 대상에 한 번만 가벼운 follow-up
- day 10-14:
  - 반응 정리 후 `package B`, `package C`, `hold`, `closed` 분기

## 4. channel guidance

- `email`:
  - 자료 링크와 안내 문구를 정리해 보내기 좋다
- `messenger`:
  - 가까운 지인형 first wave에 적합하다
- `dm`:
  - 짧은 first look 요청에는 가능하지만 자료 전달 구조는 더 간단해야 한다

## 5. follow-up message rule

- follow-up은 한 번만 한다
- 톤은 `재촉`이 아니라 `부담 없이 지나가도 된다` 쪽으로 둔다
- 답장이 없으면 `no response` 또는 `closed`로 정리한다

## 6. orchestra note

- 이 보드의 목적은 실제 이름을 많이 적는 것이 아니라, primary 3명을 정확히 매핑해 첫 파동을 시작할 수 있게 만드는 것이다
- first wave는 작게 시작하고, 반응이 좋을 때만 secondary lane이나 `package B`로 확장한다
