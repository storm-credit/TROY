# TROY External Evaluation Package Plan

## 1. purpose

이 문서는 외부 평가 전달을 목적별로 어떻게 나눌지 잠근다.

핵심 원칙:

- 전부를 한 번에 보내지 않는다
- 목적에 따라 묶음을 바꾼다
- 작품 이해 부담과 피드백 부담을 분리한다

## 2. recommended package ladder

### package A - short first look

언제:

- 아직 상대가 작품을 처음 보는 단계
- 짧게 감을 보고 싶은 단계

보내기:

- `00_START_HERE.md`
- `01_EVALUATOR_NOTE.md`
- `showcase_index.md`
- `pilot_showcase/`

목표:

- 작품 첫인상
- 정서 톤
- ending ethic
- 더 읽을지 여부 판단

### package B - full manuscript evaluation

언제:

- 샘플 반응이 괜찮고 전체 통독 판단이 필요할 때

보내기:

- `00_START_HERE.md`
- `01_EVALUATOR_NOTE.md`
- `showcase_index.md`
- `너라는운율_submission_draft_v1.md`

목표:

- 장편 구조
- 감정선 일관성
- 결말 작동 여부

### package C - feedback collection

언제:

- 실제 코멘트와 수정 의견을 회수하고 싶을 때

보내기:

- `00_START_HERE.md`
- `01_EVALUATOR_NOTE.md`
- `02_ALPHA_READER_FEEDBACK_GUIDE.md`
- `03_FEEDBACK_TEMPLATE.md`
- `04_FEEDBACK_TRIAGE_NOTE.md`
- `05_DELIVERY_CHECKLIST.md`
- `너라는운율_submission_draft_v1.md`

목표:

- 피드백 양식 통일
- 감상과 수정 의견 분리
- 후속 triage 준비

## 3. risks

- 내부 ops 전체를 통째로 보내면 과잉 산출로 보일 수 있다
- showcase와 full manuscript를 목적 없이 같이 보내면 읽기 피로가 커진다
- 첫 접점에서 wave showcase를 전부 보내는 건 추천하지 않는다

## 4. director recommendation

권장 순서:

1. `package A`
2. 반응 확인
3. `package B`
4. 실제 코멘트 필요 시 `package C`

추천 기본값:

- 첫 외부 접촉은 `package A`
- 신뢰도 높은 독자 또는 평가자에게만 `package B`
- 수정 회수 단계에서만 `package C`
