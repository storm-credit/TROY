# TROY Package A First-Wave Messages

## 1. purpose

이 문서는 `package A` 첫 외부 전달에서 바로 복붙해 쓸 수 있는 메시지 최종형을 정리한다.

기본 원칙:

- 길지 않게
- 읽어야 할 자료를 좁게 제시
- 첫 반응만 받는다
- 아직 full manuscript나 feedback form은 요구하지 않는다

## 2. default bundle

함께 보내는 자료:

- `00_START_HERE.md`
- `01_EVALUATOR_NOTE.md`
- `showcase_index.md`
- `pilot_showcase/`

공통 요청:

- 계속 읽고 싶은지
- 작품 톤이 바로 잡히는지
- 가장 오래 남는 장면 1-2개가 있는지

## 3. message for type A

대상:

- 문학형 로맨스 정독 독자

메시지:

안녕하세요. 요즘 정리 중인 장편 로맨스 프로젝트가 있는데, 짧은 샘플 묶음을 한번 봐주실 수 있을까 해서 보냅니다.

이 작품은 큰 사건보다 사람을 잘못 읽는 방식과 다시 묻는 방식, 그리고 낮고 선명하게 남는 감정의 잔향 쪽에 더 무게를 둡니다.

자료는 `00_START_HERE -> 01_EVALUATOR_NOTE -> pilot_showcase` 순서로 보시면 가장 편합니다.

짧게라도

- 문장 결이 어떤지
- 결말 방향의 낮은 종지가 설득력 있게 잡히는지
- 가장 오래 남는 장면이 무엇인지

정도만 말씀해 주셔도 큰 도움이 됩니다.

## 4. message for type B

대상:

- 웹소설 플랫폼 습관 독자

메시지:

안녕하세요. 장편 로맨스 프로젝트를 정리하고 있는데, 먼저 부담 없는 샘플 묶음만 짧게 봐주실 수 있을까 해서 보냅니다.

빠른 사건형보다는 관계 해석이 어긋나는 방식과 그걸 다시 바로잡는 감정선에 더 무게를 둔 작품입니다.

보실 때는 `00_START_HERE -> showcase_index -> pilot_showcase` 정도 순서로 보시면 편합니다.

가능하시면 짧게

- 초반부터 계속 읽고 싶은지
- 톤이 바로 잡히는지
- 다음을 누르게 만드는 장면이 있었는지

정도만 알려주셔도 정말 도움이 됩니다.

## 5. message for type D

대상:

- 완독형 일반 독자

메시지:

안녕하세요. 장편 원고를 정리하면서, 먼저 대표 샘플 몇 개를 묶어 본 자료가 있어 짧게 보여드리고 싶었습니다.

부담 없는 첫 반응을 듣고 싶어서 전체 원고가 아니라 샘플 묶음만 보냅니다.

`00_START_HERE`를 먼저 보고, 이어서 `pilot_showcase`만 보셔도 충분합니다.

편하게 보신 뒤

- 첫인상이 어떤지
- 감정 톤이 자연스럽게 잡히는지
- 계속 읽어 보고 싶은지

정도만 말씀해 주셔도 아주 큰 도움이 됩니다.

## 6. short close options

- 편하실 때 천천히 봐주셔도 괜찮습니다.
- 짧은 인상만 주셔도 충분히 도움이 됩니다.
- 부담 없이 솔직하게 말씀해 주시면 감사하겠습니다.
- 좋았던 장면과 멈칫했던 장면을 같이 알려주시면 더 도움이 됩니다.

## 7. orchestra note

- first wave 기본값은 항상 `package A`다
- 이 단계에서는 읽기 부담을 낮추는 것이 가장 중요하다
- 반응이 좋고 상대가 더 읽을 의사가 있을 때만 `package B` 또는 `package C`로 올린다
- `type C` 독자는 기본 first-wave 대상이 아니라 상세 피드백 레인에서 다루는 편이 낫다
