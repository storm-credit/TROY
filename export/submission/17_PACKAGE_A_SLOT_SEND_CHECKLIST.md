# TROY Package A Slot Send Checklist

## 1. purpose

이 문서는 `package A first wave`의 각 slot 발송 전에 실제 첨부/링크/문구가 빠지지 않았는지 마지막으로 확인하는 체크리스트다.

원칙:

- slot마다 보내는 묶음은 같다
- 차이는 `message version`과 `reader type expectation`만 있다
- 발송 직전에는 이 문서와 `13_PACKAGE_A_FIRST_WAVE_EXECUTION_BOARD.md`를 함께 본다

## 2. universal send packet

모든 primary / secondary slot 공통:

- `00_START_HERE.md`
- `01_EVALUATOR_NOTE.md`
- `showcase_index.md`
- `pilot_showcase/`

## 4. final pre-send questions

- 이 slot의 reader type이 실제로 맞는가
- 요구가 3개를 넘지 않는가
- 부담을 낮추는 문장이 들어갔는가
- package A 범위를 넘는 자료를 실수로 같이 붙이지 않았는가
- tracker와 execution board를 함께 업데이트할 준비가 되었는가

## 5. orchestra note

- first wave에서 실패하는 가장 흔한 이유는 자료가 부족해서가 아니라, 부탁 범위가 넓어지고 발송 묶음이 무거워지는 것이다
- 이 문서는 그 과적을 막는 마지막 체크 레이어다
