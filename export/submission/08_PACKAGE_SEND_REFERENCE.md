# TROY Package Send Reference

## 1. purpose

이 문서는 `package A / B / C`를 실제 외부 전달 직전에 빠르게 확인하기 위한 실행 레퍼런스다.

핵심:

- 누구에게
- 무엇을
- 어떤 기대치로
- 어떤 순서로 보낼지

를 한 장에서 바로 확인할 수 있게 한다.

## 2. package A

이름:

- short first look

추천 대상:

- 처음 보는 외부 검토자
- 지인 추천 전 1차 반응 확인
- 짧은 샘플만 보고 판단해 줄 사람

포함:

- `00_START_HERE.md`
- `01_EVALUATOR_NOTE.md`
- `showcase_index.md`
- `pilot_showcase/`

요청할 것:

- 첫인상
- 계속 읽고 싶은지
- 작품 톤과 감정선이 바로 잡히는지

권장 메시지 길이:

- 짧게
- 부담 없게

주의:

- 처음부터 full manuscript를 같이 붙이지 않는다
- wave showcase 전체를 같이 보내지 않는다

## 3. package B

이름:

- full manuscript evaluation

추천 대상:

- 이미 샘플 반응이 좋았던 사람
- 장편 전체 구조를 봐 줄 사람
- 편집/평가 관점의 통독이 가능한 사람

포함:

- `00_START_HERE.md`
- `01_EVALUATOR_NOTE.md`
- `showcase_index.md`
- `너라는운율_submission_draft_v1.md`

요청할 것:

- 장편 전체 구조
- 관계선 속도
- 결말 설득력
- 완독 후 잔향

권장 메시지 길이:

- 중간
- 읽기 포인트를 2-3개만 제시

기록 위치:

- `24_PACKAGE_B_FULL_READ_LOG.md`

주의:

- 피드백 양식까지 같이 요구하지 않아도 된다
- 먼저 읽어 보고 자유 감상을 주도록 두는 편이 좋다

## 4. package C

이름:

- feedback collection

추천 대상:

- 실제로 수정 의견까지 줄 의사가 있는 알파 리더
- 작품 개선을 함께 볼 신뢰도 높은 독자

포함:

- `00_START_HERE.md`
- `01_EVALUATOR_NOTE.md`
- `02_ALPHA_READER_FEEDBACK_GUIDE.md`
- `03_FEEDBACK_TEMPLATE.md`
- `04_FEEDBACK_TRIAGE_NOTE.md`
- `05_DELIVERY_CHECKLIST.md`
- `너라는운율_submission_draft_v1.md`

요청할 것:

- 구체적인 읽기 경험
- 헷갈린 구간
- 느리거나 빠르게 읽힌 구간
- 결말 온도 반응

권장 메시지 길이:

- 가장 자세하게
- 그래도 요구는 3-4개를 넘기지 않는다

기록 위치:

- `25_PACKAGE_C_FEEDBACK_INTAKE_LOG.md`

주의:

- 처음 접촉부터 package C를 쓰지 않는다
- 피드백 부담이 커서 응답률이 떨어질 수 있다

## 5. orchestra recommendation

기본 흐름:

1. `package A`
2. 반응 확인
3. 필요 시 `package B`
4. 실제 수정 회수 단계에서만 `package C`

현재 기본값:

- 첫 외부 접촉은 항상 `package A`
- `package B`는 `24_PACKAGE_B_FULL_READ_LOG.md`
- `package C`는 `25_PACKAGE_C_FEEDBACK_INTAKE_LOG.md`
